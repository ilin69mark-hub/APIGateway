# Запрос получения детальной новости

## Описание
Запрос получения детальной новости реализован в сервисе `api-gateway` и возвращает полную информацию о новости, включая комментарии к ней.

## Путь
```
GET /news/{id}
```

## Параметры
- `{id}` - числовой идентификатор новости (обязательный)

## Формат ответа
```json
{
  "id": 1,
  "title": "Заголовок новости",
  "content": "Содержание новости",
  "pub_time": "2023-12-01T10:00:00Z",
  "comments": [
    {
      "id": 1,
      "news_id": 1,
      "parent_id": null,
      "text": "Комментарий к новости"
    },
    {
      "id": 2,
      "news_id": 1,
      "parent_id": 1,
      "text": "Ответ на комментарий"
    }
  ]
}
```

## Структура данных
- `id` - идентификатор новости
- `title` - заголовок новости
- `content` - содержание новости
- `pub_time` - время публикации новости
- `comments` - массив комментариев к новости

## Как это работает
1. Запрос поступает в `api-gateway` на путь `/news/{id}`
2. Обработчик `newsDetailHandler` извлекает ID новости из URL
3. Создается объект базовой информации о новости
4. Выполняется запрос к `comment-service` для получения комментариев к новости
5. Формируется полный объект `NewsFullDetailed`, который включает как информацию о новости, так и комментарии
6. Ответ отправляется клиенту в формате JSON

## Технические детали
- Обработчик находится в файле `/workspace/api-gateway/main.go`, строки 204-246
- Запрос к сервису комментариев осуществляется через функцию `getCommentsForNews` (строки 248-281)
- Используется HTTP-запрос к `http://localhost:8081/comments?news_id={id}`
- В заголовках передается `X-Request-ID` для отслеживания запросов
- В случае ошибки получения комментариев возвращается новость без комментариев
