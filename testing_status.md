# Тестирование в проекте

## Состояние на 08.01.2026

### Postman тестирование
В проекте отсутствует реализация тестирования при помощи Postman. Не найдено:
- Коллекций Postman (.json файлов с тестами)
- Файлов окружений Postman
- Упоминаний Postman в документации или коде
- Автоматизированных тестов в формате Postman

### Доступные API-эндпоинты
1. APIGateway (localhost:8080):
   - GET /news - список новостей с пагинацией
   - GET /news?page=2 - вторая страница новостей
   - GET /news?s={query} - поиск новостей по заголовку
   - GET /news/{id} - детали новости с комментариями
   - POST /comment - добавление комментария

2. CommentService (localhost:8081):
   - POST /comments - создание комментария
   - GET /comments?news_id={id} - получение комментариев для новости

3. CensorService (localhost:8082):
   - POST /check - проверка текста на запрещённые слова

### Рекомендации
Для улучшения качества проекта рекомендуется создать коллекцию Postman с тестами для всех API-эндпоинтов, включая:
- Позитивные и негативные сценарии
- Проверки с разными параметрами
- Тестирование пагинации и поиска
- Проверки работы фильтрации комментариев
